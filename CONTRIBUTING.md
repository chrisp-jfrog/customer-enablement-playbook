# Contributing to the Customer Enablement Playbook

## Branching Strategy
- `main` — stable, production-ready content
- `develop` — integration branch for new content
- `feature/<name>` — individual feature or content branches

## Pull Request Guidelines
- Include a clear description of changes
- Reference any related issues or tickets
- Ensure all links and assets are valid before submitting

## Content Standards
- Use plain language; avoid jargon unless defined in `/docs/reference`
- All images must be stored in `/assets/images`
- Templates belong in `/src/templates`
